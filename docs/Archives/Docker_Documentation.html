Container Management Commands

docker ps Basic
List running containers
Shows currently active containers with their status, ports, and names. Essential for checking service health.
docker ps -a Extended
List all containers (including stopped)
Shows all containers regardless of status. Useful for debugging failed services and checking container history.
docker restart <container> Control
Restart a specific container
Stops and starts a container. Used extensively for applying configuration changes (nginx-proxy, ffcreator-service).
docker stop <container> Control
Stop a running container gracefully
Sends SIGTERM signal to container processes. Preferred method for controlled shutdown.
docker start <container> Control
Start a stopped container
Starts an existing container that was previously stopped. Maintains container configuration and data.

Container Inspection & Debugging

docker logs <container> Debug
View container logs
Essential for debugging. Used extensively to check FFCreator startup logs and error messages.
docker logs --tail 10 <container> Debug
View last 10 log entries
Quickly check recent container activity. Used frequently during troubleshooting sessions.
docker logs -f <container> Debug
Follow log output in real-time
Stream live logs for monitoring container behavior during testing and debugging.
docker inspect <container> Analysis
Get detailed container information
Returns complete JSON metadata including network settings, mounts, and environment variables. Critical for nginx-proxy analysis.
docker exec -it <container> /bin/sh Interactive
Enter container shell interactively
Access container's internal shell for file editing, process management, and direct debugging. Used extensively for FFCreator server.js modifications.
docker exec <container> <command> Execute
Execute command in running container
Run single commands without interactive shell. Used for file operations, process checks, and quick diagnostics.

File Operations & Management

docker cp <file> <container>:<path> Transfer
Copy file from host to container
Transfer configuration files into containers. Used to deploy fixed server.js to ffcreator-service.
docker cp <container>:<path> <file> Transfer
Copy file from container to host
Extract files from containers for backup or analysis. Used to backup original server.js files.
docker exec <container> cat /app/file.js View
View file contents inside container
Inspect configuration files without copying. Used extensively to examine server.js and package.json contents.
docker exec <container> ls -la /app List
List directory contents in container
Explore container file system structure. Used to understand FFCreator's directory layout and identify configuration files.
docker exec <container> grep -n "pattern" file Search
Search for patterns in container files
Find specific code patterns or configurations. Used to locate API endpoint definitions and identify duplicates.

Network Management

docker network ls Network
List Docker networks
View available networks including custom bridge networks used by docker-compose services.
docker network inspect <network> Network
Inspect network configuration
Analyze network topology and container connectivity. Used to understand n8n-ai-studio network structure.
docker exec <container> nslookup <service> DNS
Test DNS resolution between containers
Verify container name resolution. Used to test nginx-proxy connectivity to ffcreator-service.
docker exec <container> ping <service> Connectivity
Test network connectivity between containers
Verify network routing and connectivity. Essential for troubleshooting service communication issues.

Docker Compose Commands

docker-compose up -d Startup
Start all services in detached mode
Launch the entire N8N AI Studio stack. Creates networks, volumes, and starts all containers as defined in docker-compose.yml.
docker-compose restart <service> Control
Restart specific service
Restart individual services without affecting others. Used frequently for ffcreator service after configuration changes.
docker-compose logs <service> Debug
View service logs
Access logs for specific services managed by docker-compose. More convenient than using container names.
docker-compose ps Status
Show compose project status
Display status of all services in the compose project with health checks and port mappings.
docker-compose down Shutdown
Stop and remove all containers
Gracefully shutdown entire stack. Removes containers but preserves volumes and networks.

Service-Specific Commands

docker exec nginx-proxy nginx -t Nginx
Test nginx configuration syntax
Validate nginx.conf before restarting. Critical for preventing nginx startup failures after configuration changes.
docker exec ffcreator-service node -c /app/server.js Node.js
Check Node.js syntax
Validate JavaScript syntax before deployment. Used to catch syntax errors in FFCreator server.js modifications.
docker exec ffcreator-service ps aux | grep node Process
Check running Node.js processes
Monitor Node.js server processes and their resource usage. Used to verify server startup and identify process issues.
docker exec ffcreator-service pkill -f "node server.js" Process
Kill Node.js server process
Terminate specific Node.js processes for manual restart. Used when configuration changes require process restart.
docker exec ffcreator-service npm list express Dependencies
Check NPM package installation
Verify that required Node.js packages are installed. Used to confirm Express.js and other dependencies.

API Testing & Validation

curl -s http://localhost:3001/health Health
Test API health endpoint
Verify service health and get system information. Essential for monitoring FFCreator service status.
curl -s -w "Status: %{http_code}\n" http://localhost:3001/api Test
Test API endpoint with status code
Test API endpoints and capture HTTP status codes. Used extensively to verify endpoint functionality.
curl -s http://localhost:3001/ | jq '.endpoints' JSON
Extract specific JSON fields
Parse JSON responses and extract specific data. Used to verify API endpoint listings and service information.
curl -X POST http://localhost:3001/create-test-video POST
Test video creation endpoint
Test FFCreator's video generation functionality. Validates that the core video processing capability works correctly.

Troubleshooting & Maintenance

docker system prune Cleanup
Clean up unused Docker objects
Remove unused containers, networks, images, and volumes. Helps free up disk space and resolve conflicts.
docker stats Monitor
Monitor container resource usage
Real-time monitoring of CPU, memory, and network usage for all running containers.
docker exec <container> df -h Storage
Check container disk usage
Monitor filesystem usage within containers. Important for services that generate large files like FFCreator.
docker exec <container> netstat -tlnp Network
Check listening ports in container
Verify which ports are open and listening within containers. Used to confirm service port configurations.
docker volume ls Volume
List Docker volumes
View all Docker volumes for persistent data management. Important for understanding data persistence setup.
docker inspect --format='{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' <container> Network
Get container IP address
Extract specific network information using Go templates. Useful for debugging network connectivity issues.

Quick Reference - Project Commands

Common Startup Sequence

1. docker-compose up -d
2. docker ps
3. docker logs nginx-proxy --tail 10
4. curl http://localhost:3001/health

Troubleshooting Sequence

1. docker logs <service>
2. docker exec -it <service> /bin/sh
3. docker inspect <service>
4. docker restart <service>

Pro Tips

  • • Always test nginx configuration with nginx -t before restarting
  • • Use --tail flag with logs to avoid overwhelming output
  • • Keep backups of working configurations before making changes
  • • Use docker-compose commands when possible for service management
  • • Check container health with /health endpoints regularly
