## üåê Nginx Network Architecture & API Endpoints

**Network Flow:**
```
Internet ‚Üí Nginx (80/443) ‚Üí Internal Docker Network ‚Üí Services (expose only)
```

Nginx is the network wrapper - all external requests route through it:
- **External Access**: Only Nginx ports 80/443/8443 exposed
- **Internal Services**: Use expose: for container-to-container communication
- **Security**: No direct external access to ComfyUI, FFCreator, Kokoro

## üîÑ N8N Workflow Integration Pattern

**Complete Workflow Sequence:**
1. **N8N Trigger** ‚Üí Receives input data
2. **HTTP Request Node** ‚Üí POST to ComfyUI for image generation
3. **Wait/Poll Node** ‚Üí Monitor ComfyUI job completion
4. **HTTP Request Node** ‚Üí GET generated image from ComfyUI
5. **HTTP Request Node** ‚Üí POST script to Kokoro for TTS
6. **HTTP Request Node** ‚Üí GET audio file from Kokoro
7. **HTTP Request Node** ‚Üí POST combined assets to FFCreator
8. **Wait/Poll Node** ‚Üí Monitor FFCreator job completion
9. **HTTP Request Node** ‚Üí GET final video download URL
10. **Webhook Response** ‚Üí Return video to caller

**File Path Integration:**
- **ComfyUI outputs** ‚Üí `/app/ai-assets/` (mounted in FFCreator)
- **Kokoro audio** ‚Üí `/app/tts-audio/` (mounted in FFCreator)
- **FFCreator videos** ‚Üí `/app/videos/` (accessible via download endpoint)
